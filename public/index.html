Release update

GhostDrop v1.1 beta

This update focuses on improving the frontend experience, API flexibility, and overall reliability of GhostDrop.

Features

  • Added a dedicated download gateway page with a trust warning and automatic redirect.
  • Introduced file metadata support, including view tracking and a metadata endpoint.
  • Added a password-protected file deletion endpoint with initial frontend support.
  • Added a /health endpoint exposing uptime, file count, CPU usage, and memory usage.
  • Introduced optional ngrok tunnel support with configurable enable and disable controls.

Improvements

  • Refreshed the web UI with a cleaner structure and updated branding.
  • Separated JavaScript and CSS assets for better maintainability.
  • Implemented dynamic API URL resolution to adapt to backend changes.
  • Updated root API behavior to redirect users to the main hosted homepage.
  • Improved API documentation, warnings, and overall project presentation.

Fixes

  • Fixed frontend pathing issues.
  • Resolved dynamic URL handling bugs to improve reliability.

API Reference

Base URL: https://link.ghostdrop.qzz.io  ·  No auth  ·  CORS open
GET /health/ health check ▶

Returns the server information about the server such as the amount of files present the uptime the ram and cpu usage

// 200 OK { "files_stored": 1, "cpu_usage": "6.9%", "memory_usage": "63.9%", "uptime": "31.02 seconds" }
POST /upload/ upload a file ▶

Accepts multipart/form-data with a single file field. Returns a UUID and expiry metadata.

Warning: API file uploads are limited to ~4.5 MB. Larger files will be rejected.

fieldtyperequireddescription
filefileyesthe file to upload
passwordheadernooptional file password sent as a request header
curl -X POST "https://link.ghostdrop.qzz.io/upload/" \ -H "password: my-secret" \ -F "file=@example.txt"
// 200 OK { "id": "2b8c5f4c-0f1b-4d8e-8d3c-1f3a4d6b7e9f", "original_name": "example.txt", "expires_in_hours": 6 }
GET /files/{file_id} download a file ▶

Downloads a file by UUID. Serves the original filename. If the file is password protected, send the password header or the API returns 401. If expired, deletes and returns 410.

paramindescription
file_idpathUUID from POST /upload/
passwordheaderrequired only for protected files
curl -O -J "https://link.ghostdrop.qzz.io/files/{file_id}" \ -H "password: my-secret"
200 binary file stream 401 { "error": "Unauthorized" } 404 { "error": "File not found" } 410 { "error": "this file is gone" } 500 { "error": "Internal server error" }
GET /metadata/{file_id} fetch file metadata ▶

Fetches metadata for an uploaded file by UUID. Returns the original filename, expiry timestamp, current view count, and whether a password is required.

paramindescription
file_idpathUUID from POST /upload/
curl "https://link.ghostdrop.qzz.io/metadata/{file_id}"
// 200 OK { "original_name": "example.txt", "expires_at": "2026-04-20T18:30:00+00:00", "views": 3, "has_password": true }
404 { "detail": "File not found" } 410 { "error": "this file is gone" }

Privacy Policy

GhostDrop collects almost nothing. Files and their metadata are deleted automatically after 6 hours.

What we store temporarily

When you upload a file, GhostDrop holds the file contents (under a randomly generated UUID) and a metadata entry with the original filename and expiry timestamp. Both are automatically deleted after 6 hours.

What we do not collect

  • No names, email addresses, or personal identifiers
  • No user accounts or authentication data
  • No analytics or tracking cookies. A small functional cookie may be used to remember that you dismissed release notes.
  • No analytics or behavioural data
  • We do not sell or share data with third parties

File access

Files are accessible to anyone with the UUID. Treat your file ID as a secret. There are no access controls beyond link knowledge.

Server logs

The server may generate standard HTTP access logs (IP addresses, request paths) during normal operation. These are not persisted beyond the server process lifetime.

Security

Files are not encrypted at rest in the default configuration. Network transport security depends on your deployment setup.

Children's privacy

GhostDrop is not directed at children under 13. We do not knowingly collect information from minors.

Changes

We may update this policy at any time. The current version is always available here.
