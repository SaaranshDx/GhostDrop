Release update

GhostDrop v1.4

This update introduces custom slugs, password-protected uploads, QR code scanning, Docker support, and a completely reworked download page experience.

Features

  • Custom Slugs — Upload files with a custom identifier instead of a random string, with format validation and reserved slug protection
  • Password Protection — Optionally protect uploaded files with a password; stored securely using Argon2 hashing
  • File Metadata — Download page now shows file name, size, view count, and expiry time with Discord embed support
  • QR Code Scanning — Built-in QR scanner to quickly open shared file links from other devices
  • Share Popup — Dedicated share UI with direct link, QR code, and NFC sharing options
  • Client Page & Installers — New client download page with Windows install script and Android APK support
  • Debug Overlay — Developer debug panel with real-time frontend event logging
  • Docker Support — Official Dockerfile for easy self-hosted deployment

Improvements

  • Download Page — Complete overhaul with file metadata display (name, size, views, expiry) and smoother animations
  • Slug Validation — Real-time input validation with inline error messages and reserved slug list
  • Error Handling — Granular HTTP error codes (400, 409, 413) with frontend-specific error toasts and inline feedback
  • API Robustness — Fixed slug header/form mismatch and server URL resolution for reliable operation behind proxies

Fixes

  • Slug Upload — Fixed slug not being received by backend (Header → Form data migration)
  • Dev Server Leak — Fixed backend resolving to dev server URL in production builds
  • Docker Build — Fixed broken dependency file path in Docker image
  • File Overwrite — Added conflict detection: uploading with an existing slug now returns 409 instead of silently overwriting

Note: all mobile apps would be automatically updated

API Reference

Base URL: https://link.ghostdrop.qzz.io  ·  No auth  ·  CORS open
GET /health/ health check ▶

Returns the server information about the server such as the amount of files present the uptime the ram and cpu usage

// 200 OK { "files_stored": 1, "cpu_usage": "6.9%", "memory_usage": "63.9%", "uptime": "31.02 seconds" }
POST /upload/ upload a file ▶

Accepts multipart/form-data with a single file field. Returns a short file ID (either a 6-character random string or your custom slug) and expiry metadata.

Warning: API file uploads are limited to ~4.5 MB. Larger files will be rejected.

fieldtyperequireddescription
filefileyesthe file to upload
slugformnocustom identifier (2+ chars, only letters/numbers/hyphens/underscores). Must not be a reserved slug (see below).
passwordformnooptional file password for protected downloads

Reserved slugs — these cannot be used: upload, download, delete, health, files, metadata, api, admin, static, assets, public, docs, openapi, redoc, www, root, system, config, help, status

curl -X POST "https://link.ghostdrop.qzz.io/upload/" \ -F "file=@example.txt" \ -F "slug=my-file" \ -F "password=my-secret"
// 200 OK { "id": "my-file", "original_name": "example.txt", "expires_in_hours": 6 }
400 { "detail": "Slug can only contain letters, numbers, hyphens and underscores" } 400 { "detail": "That slug is reserved and cannot be used" } 400 { "detail": "Slug must be at least 2 characters long" } 409 { "detail": "Slug already in use" } 413 { "detail": "File too large" }
GET /files/{file_id} download a file ▶

Downloads a file by ID (either a custom slug or a short random string). Serves the original filename. If the file is password protected, send the password header or the API returns 401. If expired, deletes and returns 410.

paramindescription
file_idpaththe ID or slug returned from POST /upload/
passwordheaderrequired only for protected files
curl -O -J "https://link.ghostdrop.qzz.io/files/{file_id}" \ -H "password: my-secret"
200 binary file stream 401 { "error": "Unauthorized" } 404 { "error": "File not found" } 410 { "error": "this file is gone" } 500 { "error": "Internal server error" }
GET /metadata/{file_id} fetch file metadata ▶

Fetches metadata for an uploaded file by ID (slug or random string). Returns the original filename, expiry timestamp, current view count, and whether a password is required.

paramindescription
file_idpaththe ID or slug from POST /upload/
curl "https://link.ghostdrop.qzz.io/metadata/{file_id}"
// 200 OK { "original_name": "example.txt", "expires_at": "2026-04-20T18:30:00+00:00", "views": 3, "has_password": true }
404 { "detail": "File not found" } 410 { "error": "this file is gone" }
DELETE /delete/{file_id} delete a file ▶

Self-hosted only. This endpoint is not available on the public GhostDrop instance. It only works on your own deployment.

Deletes an uploaded file and its metadata by ID. If the file is password protected, the correct password must be sent as a header.

Privacy Policy

GhostDrop is designed to collect as little as possible, but it does process uploaded files, temporary metadata, and limited usage data needed to operate the service.

What we store temporarily

When you upload a file, GhostDrop stores the file contents under a short alphanumeric file ID plus metadata such as the original filename, expiry timestamp, view count, and whether the file requires a password. These items are intended to be automatically deleted after about 6 hours, or sooner if the file is removed.

Password-protected files

If you set an optional file password, GhostDrop stores a derived Argon2 password hash in metadata so the backend can verify future download attempts. GhostDrop does not need to store the plaintext password after upload.

What we do not collect

  • No names, email addresses, or personal identifiers
  • No user accounts or long-term profile data
  • No plaintext file passwords after upload
  • We do not sell or share data with third parties

Cookies and analytics

The public site may use a small functional cookie to remember that you dismissed release notes. The site also currently loads Google Analytics on the public landing page, which may collect standard usage information such as page views, browser or device data, and similar telemetry according to Google's implementation.

File access

Files are accessible to anyone with the valid file link or ID. If a file is password protected, that password is also required before download. Treat both the file link and any password as secrets.

Server logs

The server may generate standard HTTP or application logs during normal operation, which can include IP addresses, request paths, status information, and similar technical data. Retention depends on deployment and hosting configuration.

Security

Optional file passwords are hashed before storage, but files are not encrypted at rest in the default configuration. Network transport security and log retention depend on your deployment and hosting setup.

Children's privacy

GhostDrop is not directed at children under 13. We do not knowingly collect information from minors.

Changes

We may update this policy at any time. The current version is always available here.
