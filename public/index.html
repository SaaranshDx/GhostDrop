API Reference

Base URL: https://link.ghostdrop.qzz.io  ·  No auth  ·  CORS open
GET /health/ health check ▶

Returns the server information about the server such as the amount of files present the uptime the ram and cpu usage

// 200 OK { "files_stored": 1, "cpu_usage": "6.9%", "memory_usage": "63.9%", "uptime": "31.02 seconds" }
POST /upload/ upload a file ▶

Accepts multipart/form-data with a single file field. Returns a UUID and expiry metadata.

Warning: API file uploads are limited to ~4.5 MB. Larger files will be rejected.

fieldtyperequireddescription
filefileyesthe file to upload
passwordheadernooptional file password sent as a request header
curl -X POST "https://link.ghostdrop.qzz.io/upload/" \ -H "password: my-secret" \ -F "file=@example.txt"
// 200 OK { "id": "2b8c5f4c-0f1b-4d8e-8d3c-1f3a4d6b7e9f", "original_name": "example.txt", "expires_in_hours": 6 }
GET /files/{file_id} download a file ▶

Downloads a file by UUID. Serves the original filename. If the file is password protected, send the password header or the API returns 401. If expired, deletes and returns 410.

paramindescription
file_idpathUUID from POST /upload/
passwordheaderrequired only for protected files
curl -O -J "https://link.ghostdrop.qzz.io/files/{file_id}" \ -H "password: my-secret"
200 binary file stream 401 { "error": "Unauthorized" } 404 { "error": "File not found" } 410 { "error": "this file is gone" } 500 { "error": "Internal server error" }
GET /metadata/{file_id} fetch file metadata ▶

Fetches metadata for an uploaded file by UUID. Returns the original filename, expiry timestamp, current view count, and whether a password is required.

paramindescription

// 200 OK { "original_name": "example.txt", "expires_at": "2026-04-20T18:30:00+00:00", "views": 3, "has_password": true }
404 { "detail": "File not found" } 410 { "error": "this file is gone" }

GhostDrop Client

Quick and secure file sharing with GhostDrop. Right-click any file and upload it with optional password protection.

Features

Password Protection

Optional password for uploaded files. Leave blank for no protection or set a password to secure your upload.

Auto-Copy Links

Share links automatically copied to your clipboard. Paste anywhere to share your files instantly.

Context Menu Integration

Right-click any file and select "Upload with GhostDrop" for seamless integration with Windows Explorer.

Privacy & Security

Files uploaded directly to GhostDrop servers with optional password protection. Links expire based on server policy.

Installation

Quick install with a single PowerShell command. Copy and paste this into PowerShell (as Administrator) and press Enter:

iex (New-Object Net.WebClient).DownloadString('https://raw.githubusercontent.com/SaaranshDx/Ghostclient/main/installer.ps1')

Or use this alternative if the above doesn't work:

Invoke-WebRequest -Uri 'https://raw.githubusercontent.com/SaaranshDx/Ghostclient/main/installer.ps1' -UseBasicParsing | Invoke-Expression

Execution Policy Error? If you get an execution policy error, run:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

After Installation

  • Right-click any file → "Upload with GhostDrop"
  • Files are uploaded securely to GhostDrop servers
  • Instant link generation and clipboard copy
  • Works with files of any type and size (up to your system limits)

Privacy Policy

GhostDrop is designed to collect as little as possible, but it does process uploaded files, temporary metadata, and limited usage data needed to operate the service.

What we store temporarily

When you upload a file, GhostDrop stores the file contents under a short alphanumeric file ID plus metadata such as the original filename, expiry timestamp, view count, and whether the file requires a password. These items are intended to be automatically deleted after about 6 hours, or sooner if the file is removed.

Password-protected files

If you set an optional file password, GhostDrop stores a derived Argon2 password hash in metadata so the backend can verify future download attempts. GhostDrop does not need to store the plaintext password after upload.

What we do not collect

  • No names, email addresses, or personal identifiers
  • No user accounts or long-term profile data
  • No plaintext file passwords after upload
  • We do not sell or share data with third parties

Cookies and analytics

The public site may use a small functional cookie to remember that you dismissed release notes. The site also currently loads Google Analytics on the public landing page, which may collect standard usage information such as page views, browser or device data, and similar telemetry according to Google's implementation.

File access

Files are accessible to anyone with the valid file link or ID. If a file is password protected, that password is also required before download. Treat both the file link and any password as secrets.

Server logs

The server may generate standard HTTP or application logs during normal operation, which can include IP addresses, request paths, status information, and similar technical data. Retention depends on deployment and hosting configuration.

Security

Optional file passwords are hashed before storage, but files are not encrypted at rest in the default configuration. Network transport security and log retention depend on your deployment and hosting setup.

Children's privacy

GhostDrop is not directed at children under 13. We do not knowingly collect information from minors.

Changes

We may update this policy at any time. The current version is always available here.
